Java And C++

发表于 2016-10-25   |   分类于 Java   |  

虽然Title是Java And C++，但是本文可能会偏向Java。因为小时候对手机上出现的一个coffee杯，然后上面出现热气的印象很深刻，当时就感觉很酷的样子，后来才知道那就Java的标志，所以博主可能行文中间会偏向Java。

先来安利一下Java的advantage

1.Java为纯面向对象语言，它能够反映一切生活中的对象，编写程序更为容易；
2.平台无关性，”一次编译，到处运行”，程序源代码编译成字节码，然后在java虚拟机上执行；
3.Java提供了很多内置的类库，简化开发人员的程序设计工作，比如Java语言提供了对多线程的支持，提供了对网络通信的支持，以及垃圾回收器（Garbage Collection）；
4.java提供了对web应用开发的支持，applet、servlet和jsp用来开发web应用程序，socket，RMI用来开发分布式应用程序；
5.java具有较好的安全性和健壮性，防止恶意代码攻击的安全机制（数组边界检测和Bytecode校验等）；
6.去除了C++语言中难以理解、容易混淆的特性，比如头文件、指针、结构、单元、运算符重载、虚拟基础类、异常处理机制、多重继承等；

Comparison between Java and C++:

1.Java为纯面向对象语言，所有代码（报错变量、函数等）必须在类中实现，除基本数据类型（包括int、float等）外，所有类型都是类。此外，Java语言中不存在全局变量或者全局函数，而C++兼具面向过程和面向过程编程的特点，可以定义全局变量和全局函数；
2.与C++相比，Java没有指针的概念，这有效防止了C/C++语言中操作指针可能引起的系统问题，从而使程序变得更加安全；
3.与C++相比，Java语言不支持多重继承，但是Java引入了接口的概念，可以同时实现多个接口。由于接口也具有多态性，因此java语言中可以通过实现多个接口来实现与C++语言中多重继承类似的目的；
4.在C++语言中，需要开发人员去管理内存的分配（包括申请与释放），而Java语言提供了垃圾回收器来实现垃圾的自动回收，不需要程序显式地管理内存的分配。在C++语言中，通常会把释放资源的代码放到析构函数中，Java语言中虽然没有析构函数，但却引入了一个finalize()方法，当垃圾回收器将要释放无用对象的内存时，会首先调用该对象的finalize()方法，因此，开发人员不需要关心也不需要知道对象所占的内存空间何时会被释放；
5.C++语言支持运算符重载，而Java语言不支持运算符重载。C++语言支持预处理，Java语言没有预处理器，但它提供的import机制与C++中的预处理机制类似；
6.Java具有平台无关性，Java提供对注释文档的内建支持。

其实呢，博主觉得Java最大的优势就是跨平台，也就是所谓的“一次编译，到处运行”。
